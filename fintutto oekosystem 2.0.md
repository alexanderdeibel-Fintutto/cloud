📦 2\) Stripe Produkte & Preise automatisch per Code  
Du schreibst ein Setup-Skript, das Stripe Produkte \+ Preise erstellt, z. B.:

import Stripe from 'stripe';  
const stripe \= new Stripe(process.env.STRIPE\_SECRET\_KEY, { apiVersion: '2024-11-20' });

const products \= \[  
  {name: 'Fintutto Premium', price: 1000, interval: 'month'},  
  {name: 'Fintutto Biz Pro', price: 2500, interval: 'month'},  
\];

for (const p of products) {  
  const prod \= await stripe.products.create({name: p.name});  
  await stripe.prices.create({  
    product: prod.id,  
    unit\_amount: p.price,  
    currency: 'eur',  
    recurring: {interval: p.interval},  
  });  
}  
→ Dieses Skript führst du lokal oder als Deployment Step aus (z. B. CI/CD).

Du kannst so alle Stripe-Preise automatisiert definieren, nicht nur per Hand im Dashboard.

⚡ 3\) Edge Functions in Supabase für Stripe Integration  
Du nutzt Supabase Edge Functions (Deno/Typescript):

supabase functions new create-checkout  
👉 Funktion erstellt Stripe Checkout Sessions  
👉 Funktion verarbeitet Webhook-Events  
👉 Du speicherst Subscription-Status in Supabase

Supabase Edge Functions sind dafür ideal, weil sie serverseitig laufen und Stripe-Keys sicher halten. 

📍 4\) Stripe Webhooks empfangen  
Stripe sendet Events (z. B. checkout.session.completed).  
Du legst folgende Edge Function an:

supabase functions new stripe-webhook  
Im Code:

import Stripe from 'stripe';  
const stripe \= new Stripe(env.STRIPE\_SECRET\_KEY, {apiVersion: '2024-11-20'});

export default async (request) \=\> {  
  const payload \= await request.text();  
  const sig \= request.headers.get('Stripe-Signature')\!;  
  const event \= stripe.webhooks.constructEvent(payload, sig, env.STRIPE\_WEBHOOK\_SECRET);

  // z. B. Zahlung abgeschlossen → in Supabase markieren  
};  
Dann im Stripe Dashboard → Webhooks → URL deiner Function einfügen.

⚠️ Das ist der Standard-Flow, deshalb brauchst du Apps & Functions, nicht nur CLI. 

📈 5\) Stripe Daten synchronisieren (optional)

compute\_features.py  
import numpy as np

def compute\_state(transactions):  
    inflow \= \[t\["amount"\] for t in transactions if t\["direction"\] \== "inflow"\]  
    outflow \= \[t\["amount"\] for t in transactions if t\["direction"\] \== "outflow"\]

    income\_mean \= np.mean(inflow) if inflow else 0  
    expense\_mean \= np.mean(outflow) if outflow else 0

    burn\_rate \= expense\_mean \- income\_mean

    return {  
        "income\_mean": income\_mean,  
        "expense\_mean": expense\_mean,  
        "burn\_rate": burn\_rate  
    }
